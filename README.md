# 会議関所（かいぎせきしょ） — Meeting Gatekeeper

**「会議を開く」前に、まず関所を通る。** 社内会議の効率化を、時間・人数・報連相の3軸で統合した新発想のシステムです。

デモ: `index.html` をブラウザで開くだけで動作します（サーバー・インストール不要、データはブラウザ内に保存）。

## コンセプト

1. **会議は1分でも短い方がいい** — コストは「積み上がる」のではなく「予算から減っていく」
2. **会議は参加者が少なければ少ないほどいい** — 「聞くだけ」の参加者はゼロにする
3. **報連相の統合** — 報告（過去）はレポートで、連絡（現在）はチャットで、相談（未来）だけが会議になる

## 従来ツールとの違い（何が画期的か）

### 1. 報連相ルーター — 会議は「最後の手段」

従来の会議ツールは「会議を開くこと」が前提です。本システムは逆で、**会議を開かせないことから始まります**。

伝えたい内容が「過去・現在・未来」のどれかを最初に仕分けし、

| 時制 | 種別 | 出口 |
|------|------|------|
| 過去のこと | 報告 | レポートのテンプレートを自動生成（会議は開けない） |
| 現在のこと | 連絡 | チャット投稿文を自動生成（会議は開けない） |
| 未来のこと | 相談 | ここで初めて会議の設計へ進める |

報告・連絡に仕分けられた場合は「本来開いていたはずの会議」の想定コストを**回避コスト**として記録します。開かなかった会議の節約額が積み上がる、というのが本システムの中心的な指標です。

### 2. 「未来の問い」しか議題にできない

議題は必ず**疑問形（〜するか？〜すべきか？）**でなければ登録できません。
「〜の報告」「〜の共有」という議題は文法的に弾かれ、レポート／チャットへ誘導されます。
相談とは未来への問いであり、会議の成果物は「決定」だけだからです。

### 3. 「聞くだけ」参加者の自動排除

参加者は全員に役割（決める人・意見を出す人・情報を持っている人）が必要です。
「聞くだけ」を選んだ人は招集リストから自動的に外され、**会議後レポートの送付先**に回ります。
「呼ばない代わりに、決定事項が自動で届く」ことで、安心して人数を減らせます。

### 4. コストのダウンカウンター（予算制）

参考にした[会議コストカウンター](https://s3-ap-northeast-1.amazonaws.com/mtg.design4all.biz/meeting_cost_counter.html)はコストが**増えていく**表示でしたが、本システムは会議設計時に確定した**予算がゼロに向かって減っていきます**。

- 残高がゼロに近づく緊張感が「1分でも短く」を後押し
- 予算超過は赤字表示で「超過額」がリアルタイムに積み上がる
- 議題ごとのタイムボックスも同時にカウントダウン

### 5. 決定しないと議題を閉じられない

各議題は「決定事項」を記録しないと次へ進めません。決定の記録がそのまま自動レポートになり、呼ばなかった人へ共有されます。**相談（会議）の出力が報告（レポート）になる**ことで、報連相のサイクルが1つのツール内で閉じます。

### 6. 節約ダッシュボード

- 回避した会議の件数と回避コスト
- 開いた会議の「予算 − 実績」による短縮節約額
- 会議に呼ばずレポート送付に回した延べ人数

「使った額」ではなく「取り返した時間とお金」を組織の成果として可視化します。

## 使い方

1. **関所（トリアージ）**: 伝えたいことの時制を選ぶ
2. **報告/連絡なら**: テンプレートをコピーして終わり。回避コストが記録される
3. **相談なら**: 議題（疑問形）・タイムボックス・役割つき参加者・人時単価を設定 → 値札を確認して開始
4. **コックピット**: 予算残高と議題タイマーを見ながら進行。議題ごとに決定を記録
5. **終了**: レポートが自動生成されるので、送付先リストの人へコピーして共有

## 技術構成

- 単一の `index.html`（HTML/CSS/JavaScript、依存ライブラリなし）
- データ保存は `localStorage`（人時単価・累計統計・直近レポート）
- 社内ポータルやファイルサーバーに置くだけで全員が使えます
